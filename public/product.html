Use Cases

Built for the disconnected world.

Any application where writes must succeed without a live server connection and data must converge correctly when devices reconnect.

Industrial / IoT

Sensors and gateways

Sensors writing readings at the edge. Network partitions are normal. Data lands in SQLite first, syncs when connectivity allows. HLC ordering keeps time-series data coherent across devices without trusting wall clocks.

SDK Design

Zero-config quickstart.

1. Install

# Requires Node.js 22+ (uses built-in node:sqlite)
npm install edge-native-db

2. Connect and write

import { EdgeDB } from 'edge-native-db';

const db = new EdgeDB({
  nodeId:      'field-unit-42',
  dbPath:      './data/field.db',
  upstreamUrl: 'https://sync.example.com',
  mergeSchemas: {
    tasks: { title: 'text-merge', done: 'max', updated_at: 'max' },
  },
});

db.connect();

type Task = { id: string; title: string; done: boolean; updated_at: number };
const tasks = db.collection<Task>('tasks');

tasks.upsert({ id: 't1', title: 'Inspect valve 7', done: false, updated_at: Date.now() });

const unsub = tasks.subscribe(items => console.log(`Tasks: ${items.length}`));

3. Sync control

db.sync.force();          // flush immediately
db.sync.pause();           // suspend background loop
db.sync.setOffline(true); // test offline mode
db.sync.status             // { pendingOps, syncState, connected, ... }

4. Server

import { createSyncServer } from 'edge-native-db/server';
createSyncServer(3001); // POST /sync + GET /health

5. Introspection

db.inspect.queue()           // per-op lifecycle state
db.inspect.syncLog()         // cycle history with durations
db.inspect.errors()          // failures with timestamps
db.inspect.conflicts()       // full conflict history
db.inspect.previewMerge(id)  // see merged result before applying
db.inspect.applyAutoMerge(id)// apply CRDT resolution

CRDT merge strategies

'text-merge'  // "Alice | Bob"  — sorted, joined by |
'max'         // max(local, remote) — counters, timestamps
'set-union'   // union of JSON arrays — tags, permissions
'lww'         // Last-Write-Wins (default for unlisted fields)

Deployment

From laptop to fleet.

Centralized server

Single sync server on any Node.js host. Clients connect over HTTPS.

docker run -p 3001:3001 \
  -v ./data:/data \
  edge-native-db/server

Multi-region

One sync server per region. Regional servers sync to each other via the same POST /sync protocol.

nodeId: 'server-eu-west-1'
upstreamUrl: 'https://sync-us.example.com'

LPWAN / intermittent

IoT gateways with GPRS or LoRaWAN. The oplog is durable — no data is lost if the device is offline for days.

SYNC_INTERVAL_MS=60_000
CYCLE_RETRY_DELAYS=[120s, 300s, 600s]

Air-gapped / local

Devices sync on a local LAN. A lightweight gateway node runs the sync server.

upstreamUrl: 'http://192.168.1.10:3001'
Limitations

What it cannot do yet.

Current limitations
  • No browser storage — node:sqlite is Node-only. OPFS planned for v0.2.
  • No end-to-end encryption — TLS on the wire only.
  • CRDT schema is global per entity type, not per record.
  • Sync server is single-node SQLite, no horizontal scaling.
  • No schema migrations — entity payloads are schemaless JSON.
  • HLC drift limit of ~300s — use NTP.
  • No partial sync — every device receives all changes.
  • No binary or blob support — payloads are JSON.
Known trade-offs
  • text-merge produces "Alice | Bob" — not suitable for rich text. Use Yjs or Automerge for document content.
  • Last-Write-Wins silently discards the losing write for fields without a CRDT strategy.
  • No optimistic rollback — local writes commit immediately.
  • SQLite serializes writes — use separate DB files per entity namespace for high-throughput IoT.
  • Op log grows indefinitely — no compaction or GC yet.
Roadmap

What is next.

v0.2 — Storage and browser
  • Browser adapter — OPFS with IndexedDB fallback
  • React hooks: useCollection(), useSyncStatus()
  • Op log compaction and tombstone GC
  • Structured query: collection.where()
  • Encryption at rest via SQLite cipher
v0.3 — Network and scale
  • WebSocket transport for lower latency
  • Server-sent subscriptions per entity type
  • Partial sync — device declares which types to replicate
  • mDNS peer discovery for LAN-only deployments
  • Server clustering via CRDTs between nodes
v0.4 — CRDT and merge
  • Yjs and Automerge adapter for rich-text fields
  • Custom merge callback: (local, remote) => merged
  • Per-entity merge schema override
  • Conflict preview UI as embeddable web component
v0.5 — Production hardening
  • JWT auth middleware on sync server
  • Row-level access control per device
  • Rate limiting and backpressure
  • OpenTelemetry instrumentation
  • Signed ops to prevent forged nodeId
